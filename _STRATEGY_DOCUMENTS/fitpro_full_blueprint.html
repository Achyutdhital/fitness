FitPro full project blueprint — conversion strategy, onboarding, and Django + React architecture

Finalized plans — locked

Starter — $0Points economy, ad-supported, beginner plans only, basic BMI
Flex — $9.99Unlimited library, ML body insights, health filters, no AI
Pro — $24.99Full AI chat coach, AI meal planner, weekly ML adaptation, no sessions
Elite — variesPro + assigned trainer, 2×30min sessions/mo, 10 msgs/week, video call in-platform
Custom — Elite+Elite base + extra sessions add-on, trainer gets 80%, platform 20% of add-on

What converts a free user — the full funnel

Signup hook (first 60 seconds)
1
Instant body profile — height, weight, age, sex, goal. Show ML result immediately (body fat %, ideal weight). Free users taste ML value right away.
2
Show them exactly 1 plan they can unlock for free (100 pts). Lock others behind blur with upgrade prompt.
3
Progress bar showing "You are 4 ads away from your next plan" — gamified, drives daily return.
4
Email capture on signup with welcome drip sequence.
Conversion triggers
A
After completing week 1 — "Your progress is being tracked. Upgrade to Pro to get AI-adjusted plan next week."
B
After 4 ads watched in a day — "You've hit your daily limit. Flex removes this forever for $9.99."
C
Show a locked AI meal plan preview with blurred content. "Unlock with Pro."
D
After 2 weeks on Pro — "Your trainer match is ready. See who'd be your Elite coach."
E
Trainer profile teaser on Pro dashboard — name, photo, specialty. Locked. Drives Elite upgrade.
Retention mechanics (stops churn)
Daily streak system Weekly challenges Progress photos + body trend Smart push notifications Session reminders (Elite) Weekly AI progress email Community feed (optional) Referral bonus points

Django backend — full project structure

fitpro/
├── config/ # project settings
│ ├── settings/
│ │ ├── base.py # shared settings
│ │ ├── dev.py # local dev overrides
│ │ └── prod.py # production (env vars)
│ ├── urls.py # root URL router
│ ├── wsgi.py
│ └── asgi.py # WebSocket + HTTP

├── apps/
│ ├── users/ # auth, profiles, subscriptions
│ │ ├── models.py # User, UserProfile, BodyStats
│ │ ├── serializers.py
│ │ ├── views.py # register, login, profile CRUD
│ │ └── signals.py # post-save → create profile

│ ├── subscriptions/ # plans, billing, points
│ │ ├── models.py # Plan, Subscription, PointLedger
│ │ ├── views.py # upgrade, cancel, ad-watch reward
│ │ └── stripe_webhooks.py# payment events

│ ├── plans/ # workout + meal plan library
│ │ ├── models.py # WorkoutPlan, MealPlan, Exercise, Meal
│ │ ├── views.py # list, detail, unlock with points
│ │ └── filters.py # goal/level/condition filters

│ ├── ai/ # AI chat + meal generation
│ │ ├── views.py # chat endpoint, meal plan generate
│ │ ├── prompts.py # system prompt templates
│ │ └── rate_limiter.py # per-user token limits

│ ├── ml/ # ML models + inference
│ │ ├── body_analysis.py # body fat %, BMR, ideal weight
│ │ ├── plan_ranker.py # ML plan recommendation engine
│ │ └── progress_model.py # weekly adaptation logic

│ ├── trainers/ # trainer management
│ │ ├── models.py # Trainer, TrainerAssignment, Availability
│ │ ├── views.py # assign, swap, capacity check
│ │ └── assignment.py # auto-assign logic

│ ├── sessions/ # video call + chat
│ │ ├── models.py # Session, Message, MessageQuota
│ │ ├── views.py # schedule, join, message CRUD
│ │ └── consumers.py # Django Channels WebSocket

│ ├── progress/ # tracking + analytics
│ │ ├── models.py # WorkoutLog, WeightLog, PhotoLog
│ │ └── views.py # log entry, trend data

│ ├── ads/ # ad serving + point rewards
│ │ ├── models.py # AdView, DailyAdCap
│ │ └── views.py # verify ad watch → award points

│ └── notifications/ # push + email
│ ├── tasks.py # Celery: streak alerts, upgrade nudges
│ └── email.py # drip campaigns via SendGrid

├── ml_models/ # trained model files (.pkl, .onnx)
├── media/ # user uploads (progress photos)
├── static/
├── requirements.txt
├── manage.py
└── docker-compose.yml

Tech stack

BackendDjango 5 · DRF · Django Channels (WebSocket) · Celery + Redis · PostgreSQL · JWT auth
FrontendReact 18 · Redux Toolkit · React Query · Tailwind CSS · Vite
InfrastructureDocker · Nginx · AWS S3 (media) · Stripe (billing) · Daily.co (video) · SendGrid (email)
AI / MLOpenAI API (chat coach) · Scikit-learn (body ML) · Celery tasks (weekly adaptation)
AdsGoogle AdSense or custom ad server · server-side ad view verification before awarding points
DevOpsGitHub Actions CI/CD · Docker Compose (local) · AWS EC2 or Railway (prod)
